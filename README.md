I follow the clue. When a system does something I didn't explicitly program -- when the pieces connect in a way that surprises me -- that's the signal I'm on the right track. I design with metaphors that have to survive contact with real engineering. If the metaphor breaks, the architecture was wrong. If it holds, I build it deeper. I don't work fast. I work until it's right.
